# junbit.dev 블로그 참여 개선안

> 작성일: 2026-03-26
> 현재 상태: D+30 기준, 12개 포스트, 유입 100% direct, 평균 참여시간 45초

## 1. 검색 유입 확보 (SEO)

현재 유입의 100%가 direct입니다. 구글/네이버 검색 유입이 전혀 없는 상태이므로 SEO 기본기를 세팅하는 것이 최우선 과제입니다.

### 1-1. Google Search Console & Naver 웹마스터 등록

| 작업 | 상세 |
|------|------|
| Google Search Console 등록 | `junbit-blog.pages.dev` 소유 확인 후 sitemap 제출 (`/sitemap-index.xml`) |
| Naver Search Advisor 등록 | https://searchadvisor.naver.com 에서 사이트 등록, sitemap 및 RSS 제출 |
| Bing Webmaster Tools | Google 연동으로 간단 등록 가능 |

> Astro sitemap 플러그인이 이미 설치되어 있고 `robots.txt`에 sitemap 경로도 명시되어 있으므로, **등록만 하면 됩니다.**

### 1-2. 검색 의도를 반영한 제목 개선

현재 제목들이 설명적이지만 **검색 키워드**가 약합니다. 사용자가 실제로 검색할 법한 문구를 제목 앞쪽에 배치해야 합니다.

**개선 원칙:**
- 제목 앞쪽에 모델명 + 구체적 숫자/키워드 배치
- 60자 이내 (검색 결과에서 잘리지 않도록)
- "방법", "가이드", "비교", "벤치마크" 같은 검색 의도 키워드 포함

**예시:**
| Before | After |
|--------|-------|
| `로컬 LLM을 실용적으로 쓰는 사례: 일일 주도주 분석 자동화 시스템 공개` | `Qwen3.5 9B 로컬 LLM으로 주식 분석 자동화하는 방법` |
| `MLX 캐시 관리와 컨텍스트 길이 교훈` | `Mac Mini M4 MLX 캐시 관리 가이드 - 컨텍스트 길이 최적화` |

### 1-3. 메타 description 최적화

`BaseHead.astro`의 description 메타 태그는 이미 구현되어 있지만, 각 포스트의 `description` 필드를 **150자 이내의 검색 친화적 요약**으로 다듬어야 합니다.

**체크리스트:**
- [ ] 핵심 키워드가 description 앞쪽 50자 안에 포함되는지
- [ ] "~하는 방법", "~비교", "~벤치마크 결과" 같은 액션 키워드가 있는지
- [ ] 각 포스트의 description이 고유한지 (중복 없는지)

### 1-4. 구조화된 데이터 (JSON-LD) 추가

`BaseHead.astro`에 BlogPosting 스키마를 추가하면 검색 결과에서 리치 스니펫으로 노출될 수 있습니다.

**구현 위치:** `src/components/BaseHead.astro`

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "BlogPosting",
  "headline": "{title}",
  "description": "{description}",
  "datePublished": "{pubDate}",
  "dateModified": "{updatedDate}",
  "author": {
    "@type": "Person",
    "name": "junbit",
    "url": "https://junbit-blog.pages.dev/about"
  },
  "publisher": {
    "@type": "Organization",
    "name": "junbit.dev"
  }
}
</script>
```

### 1-5. 영문 slug 유지 확인

현재 slug가 영문 기반(`qwen-3-5-27b-optimization` 등)으로 되어 있어 좋습니다. 앞으로도 이 패턴을 유지하세요. 한글 slug는 URL 인코딩으로 깨져서 공유하기 어렵습니다.

## 2. 체류시간 개선 (45초 -> 2분+ 목표)

기술 블로그 평균 참여시간 2~4분 대비 45초는 너무 짧습니다. 독자가 원하는 정보를 빠르게 찾고 더 오래 머물게 하는 구조가 필요합니다.

### 2-1. 자동 목차(TOC) 컴포넌트 추가

**가장 효과가 큰 개선 항목입니다.**

현재 `BlogPost.astro` 레이아웃에 TOC가 없어서, 긴 기술 글에서 독자가 원하는 섹션을 찾지 못하고 이탈합니다.

**구현 방안:**
- `TableOfContents.astro` 컴포넌트 신규 생성
- Astro의 `getHeadings()` API를 활용하여 H2/H3 기반 목차 자동 생성
- 포스트 상단(본문 시작 전)에 고정 배치
- 스크롤 시 현재 위치 하이라이트 (선택)

**배치 위치:** `src/layouts/BlogPost.astro`에서 `<slot />` 바로 위

```astro
---
// BlogPost.astro 내부
const { headings } = Astro.props;
---
<TableOfContents headings={headings} />
<slot />
```

### 2-2. TL;DR 섹션 템플릿화

모든 기술 포스트 상단에 3줄 요약을 넣으면 독자가 "이 글이 내가 찾던 것인지" 빠르게 판단할 수 있고, 관련 있다고 판단하면 끝까지 읽습니다.

**포스트 작성 컨벤션으로 추가:**
```markdown
> **TL;DR**
> - Mac Mini M4에서 Qwen3.5 27B를 MLX로 돌리면 초당 15토큰 나옴
> - 메모리 32GB에서 Q4 양자화 기준 안정적 운용 가능
> - 벤치마크 결과와 최적화 설정값 공유
```

**스타일링:** `global.css`에 TL;DR blockquote 스타일 추가 (배경색 구분, 왼쪽 보더)

### 2-3. 읽기 예상 시간 표시

포스트 상단에 "약 5분 읽기" 같은 표시를 추가하면 독자의 기대치를 설정하고 이탈을 줄입니다.

**구현:** 한국어 기준 분당 500자로 계산, `BlogPost.astro`의 날짜 옆에 표시

### 2-4. 관련 포스트 추천

포스트 하단(댓글 위)에 같은 태그를 가진 관련 글 2~3개를 보여주면 페이지뷰와 체류시간이 동시에 늘어납니다.

**구현 방안:**
- `RelatedPosts.astro` 컴포넌트 생성
- 현재 글의 태그와 겹치는 태그가 많은 순으로 정렬
- 최대 3개 표시

## 3. 유통 채널 확보 (SNS/커뮤니티)

현재 X에서 2세션뿐이라 글을 써도 확산이 안 되는 구조입니다.

### 3-1. 소셜 공유 버튼 추가

포스트 하단에 원클릭 공유 버튼을 추가합니다.

**대상 플랫폼:**
- X (Twitter) - 제목 + URL 자동 구성
- 카카오톡 공유 (한국 독자 타겟)
- 링크 복사 버튼

**구현 위치:** `src/layouts/BlogPost.astro`에서 본문과 댓글 사이

> OG 태그가 이미 잘 구현되어 있으므로, 공유 시 미리보기가 정상적으로 표시됩니다.

### 3-2. X(Twitter) 공유 전략

글을 발행할 때마다 X에 **한 줄 요약 쓰레드** 형식으로 공유:

```
Mac Mini M4에서 Qwen3.5 27B 돌려봤습니다.

결론부터: Q4 양자화로 초당 15토큰, 32GB RAM에서 안정 운용.

자세한 벤치마크와 설정값 ->
https://junbit-blog.pages.dev/blog/...

#LocalLLM #MLX #MacMini #Qwen
```

**포인트:**
- 결론을 먼저 말하고 링크는 마지막에
- 영문 해시태그 사용 (해외 유입 가능성 활용)
- 숫자/벤치마크 결과를 포함하면 RT가 잘 됨

### 3-3. 커뮤니티 공유 루틴

| 채널 | 공유 방식 | 빈도 |
|------|-----------|------|
| 카카오 로컬 LLM 오픈채팅 | 새 글 발행 시 링크 + 한 줄 요약 | 글 발행마다 |
| Reddit r/LocalLLaMA | 영문 요약 + 벤치마크 결과 위주 | 벤치마크/최적화 글 한정 |
| GeekNews (긱뉴스) | 한국 개발자 커뮤니티, 흥미로운 글 제출 | 월 1~2회 |
| Hacker News | 영문 제목으로 제출 | 임팩트 큰 글 한정 |

### 3-4. Footer 소셜 링크 정리

현재 Footer에 Mastodon, Twitter, GitHub 링크가 Astro 템플릿 기본값으로 되어 있습니다. **실제 사용하는 계정으로 교체**하거나 사용하지 않는 링크는 제거하세요.

## 4. 콘텐츠 전략

### 4-1. "숫자가 들어간 최적화/벤치마크 글" 패턴 유지

Qwen3.5 27B 최적화 글이 조회수 1위인 이유:
- 검색 의도에 정확히 부합 (모델명 + 하드웨어 + 성능)
- 커뮤니티에서 공유하기 좋은 구체적 숫자
- 동일 환경을 가진 사람이 직접 따라할 수 있는 실용성

**앞으로의 콘텐츠 공식:**
```
[모델명] + [하드웨어] + [구체적 성능 수치] + [재현 가능한 설정]
```

**예시 주제:**
- "Llama 4 Scout Mac Mini M4 벤치마크 - MLX vs llama.cpp 속도 비교"
- "Qwen3 32B Q4 vs Q8 양자화 품질 차이 실측"
- "Mac Mini M4 32GB로 동시에 돌릴 수 있는 LLM 조합 테스트"

### 4-2. 시리즈 글 구조 도입

단발성 글보다 시리즈 글이 재방문을 유도합니다. 현재 Qwen 관련 글이 여러 개 있지만 시리즈로 묶여 있지 않습니다.

**구현 방안:**
- frontmatter에 `series` 필드 추가 (선택)
- 시리즈 글 상단에 "이 시리즈의 다른 글" 네비게이션 표시
- 예: "Mac Mini M4 로컬 LLM 시리즈" → 1편 셋업, 2편 최적화, 3편 실전 활용

### 4-3. 영문 포스트 병행 검토

해외 도시(Paris, Tokyo, Tehran, Ulaanbaatar)에서 유입이 있다는 것은 영어권 커뮤니티 노출 가능성을 의미합니다.

**단기:** 인기 글의 영문 버전을 별도 포스트로 작성
**중기:** `/en/` 경로에 영문 블로그 구성 (Astro i18n 활용)
